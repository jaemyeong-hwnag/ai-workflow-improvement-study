새로운 AI 워크플로우 시도 "$ARGUMENTS"를 분석하고 문서화한다.

## 작업 순서

1. `docs/` 폴더의 기존 문서들을 읽어 현재까지의 분석 맥락을 파악한다.
2. 시도 내용이 코드베이스나 파일 형태로 있다면 해당 파일들을 탐색하고 읽는다.
3. 관련 논문, 공식 문서, 레퍼런스를 조사한다 (WebSearch/WebFetch 활용).
4. 다음 기준으로 분석한다:
   - 시도의 목적과 설계 의도
   - 실제 구현 방식 (코드 구조, 패턴)
   - 실행 데이터 또는 결과가 있다면 포함
   - 기존 문서와의 비교 (개선점 또는 중복)
   - 관련 논문/벤치마크 근거
   - 한계점 및 개선 가능성
5. 다음 번호의 문서 파일을 생성한다: `docs/0N_<slug>.md`
   - 기존 파일 번호를 확인하고 다음 번호 사용
   - 파일명 slug는 내용을 요약하는 영문 소문자 + 언더스코어
6. 문서 작성 후 `README.md`의 문서 구조 섹션을 업데이트한다.
7. **민감정보 검사**: 작성된 문서에 아래 항목이 포함되어 있으면 즉시 제거한다:
   - 실제 회사명, 레포명, 도메인명
   - API 키, 토큰, 크리덴셜
   - 내부 서비스명, 팀명, 개인식별정보
   - 실제 경로 (`/Users/실제이름/...` 등)

## 문서 형식

```markdown
# [시도 이름]

## 개요
[한 줄 요약]

## 배경 및 목적
[왜 이 시도를 했는지]

## 구현 방식
[코드 구조, 설계 패턴, 핵심 메커니즘]

## 실행 데이터 / 결과
[실제 결과가 있다면 수치/데이터로]

## 관련 레퍼런스
[논문, 공식 문서, 벤치마크]

## 평가
[효용성, 한계, 기존 방식 대비 비교]

## 결론
[계속 발전시킬 것인지, 폐기할 것인지, 조건부 사용인지]
```
