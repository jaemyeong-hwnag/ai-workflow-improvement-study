이 레포의 모든 마크다운 문서에서 민감정보를 전수 검사하고 제거한다.

## 검사 대상 파일

- `README.md`
- `docs/*.md`
- `.claude/commands/*.md`
- 기타 텍스트 파일 (`.txt`, `.json` 등)

## 민감정보 분류 및 처리

### 1. 식별자 (회사/레포/서비스명)
- 실제 회사명, 조직명, 팀명 → `your-company`
- 실제 레포지토리명 → `your-repo`
- 실제 서비스/도메인명 → `your-service` 또는 `your-domain`
- 실제 패키지명 (`com.실제이름.*`) → `com.yourcompany.*`

### 2. 크리덴셜
- API 키, 토큰, 시크릿 (패턴: `sk-`, `ntn_`, `figd_`, `Bearer `, `xox`, `ghp_` 등)
- 비밀번호, 해시값처럼 보이는 문자열
- → 완전 제거 또는 `<REDACTED>` 로 교체

### 3. 개인정보
- 실제 사람 이름 (작성자, 담당자 등)
- 이메일 주소
- 실제 파일시스템 경로 (`/Users/실제이름/`, `/home/실제이름/`)
- → `your-name`, `your-email`, `workflow/` 등으로 교체

### 4. 내부 비즈니스 정보
- 실제 기능명, 도메인 특화 용어 (업무 맥락에서 식별 가능한 것)
- 내부 엔드포인트 경로
- → 일반화된 표현으로 교체 (예: `your-feature`, `/your-domain/endpoint`)

## 작업 절차

1. Grep으로 패턴 검색 (이메일, 경로, 토큰 패턴 등)
2. 각 파일을 직접 읽어 맥락 기반으로 민감정보 판단
3. Edit 툴로 제거/치환
4. 수정 후 다시 읽어 누락된 것이 없는지 재확인
5. 수정한 파일 목록과 변경 내용을 요약 보고

## 판단 기준

- **제거 대상**: 외부에 공개되면 안 되는 실제 고유 식별자
- **유지 대상**: 일반적인 기술 용어, 오픈소스 라이브러리명, 논문 저자명
- **불확실한 경우**: 제거하는 방향으로 판단
